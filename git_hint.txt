Задать имя пользователя и адрес электронной почты
    git config --global user.name "tara routray"    # Изменение имени
    git config --global user.email "dev@tararoutray.com"    # изменение почты

Инициализация репозитория
    git init    # Создать пустой репозиторий Git или вновь инициализировать существующий

Создание новой ветки и переход в неё
    git branch new_branch_name

Переключение на новую ветку
    git checkout -b new_branch_name

Просмотр списка веток
    git branch

Удаление ветки
    git branch -d existing_branch_name

Добавление отдельных файлов или всех файлов в область подготовленных файлов
    git add somefile.js
    git add .

Проверка статуса репозитория
    git status

Коммит через консоль
    git commit -m "Your short summary about the commit"

Просмотр истории коммитов с изменениями
    git log -p

Просмотр заданного коммита
    git show 1af17e73721dbe0c40011b82ed4bb1a7dbe3ce29

Просмотр изменений до коммита
    git diff
    Для просмотра подготовленных изменений необходимо добавить флаг --staged
Для просмотра подготовленных изменений в файле
    git diff somefile.js

Удаление отслеживаемых файлов из текущего рабочего дерева
    git rm dirname/somefile.js

Переименование файлов
    git mv dir1/somefile.js dir2
    # указывается источник source и назначение destination.
    Источник — реально существующий файл или папка, а назначение — существующая папка

Отмена подготовленных и неподготовленных изменений
    git checkout somefile.js
    Восстановить подготовленный файл рабочего дерева
        git reset HEAD somefile.js
    для всех подготовленных файлов
    git reset HEAD

Изменение последнего коммита
    git commit --amend -m "Updated message for the previous commit"

Откат последнего коммита
    git revert HEAD    отменяет изменения, записанные только одним коммитом
    git reset откатывает проект к более раннему состоянию, удаляя все последующие коммиты

Откат заданного коммита
    git revert 1af17e

Слияние двух веток
    git merge existing_branch_name

Прекращение слияния при конфликте
    git merge --abort    остановить процесс слияния и вернуть состояние, с которого этот процесс был начат
    git reset       восстановить конфликтующие файлы до стабильного состояния

Отправка изменений в удалённый репозиторий
    git push origin main

Получение изменений из удалённого репозитория
    git pull

Слияние удалённого репозитория с локальным
    git merge origin

Отправка новой ветки в удалённый репозиторий
    git push -u origin new_branch

Использование перебазирования
    git rebase branch_name
    Перебазирование — это процесс объединения или перемещения последовательности коммитов на новый родительский снимок.
